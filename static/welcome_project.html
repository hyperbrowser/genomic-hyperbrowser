hdGSuite HyperBrowser: A web platform for high dimensional genomic data analysis

description

High dimensional genomic data

High dimensional genomic data can be presented as a matrix, with each column representing levels of samples/groups (for example, a patient, a cell type, genotype, an experimental condition and so on), and each row representing a genomic tracks, which is represented by collection of features (for example, genes, genomic locus and so on).

Here, we represented high dimensional genomic data in a GSuite format, which is a format of representing collection of genomic tracks. To read more about it look at GSuite specification. You can also look at the below example, where every single genomic track is represented by the set of characteristic defined by selected columns (genotype and mutation type) [or where each characteristics of genomic tracks are represneted by columns ( i.e. genotype and mutation type)].

This way of representing data allow for effective analysis of collections of single genomic data as well as analysis within levels of groups and among levels of groups of genomic data.

Depending on the biological question, one can divide the same set of data into different groups. Let assume that one is interested in the following statistic: average number of genomic elements per defined group, where the group is defined by the genotype (look at the left figure) or the group is defined by combination of genotype and mutation type (look at the right figure).

For the genotype groups, average number of genomic elements is counted for eta and iota, respectively. So for the given example, we calculated the number of genomic elements for tracks 1-3 separately and averaged them out (eta group), and we did the same procedure for the group iota. For the groups defined by the combinations of genotype and mutation type, we calculated a number of genomic elements for tracks 1-2 separately and averaged them out (eta-CA group), and we did the same for the other groups (eta-CG, iota-CA, iota-CG, iota-CT).

Note:

This is a specialized version of The Genomic Hyperbrowser, focusing on new functionality for analyzing collections of genomic track. More information on the general HyperBrowser functionality can be found on the main project site.

Contact information

If you have any questions, requests or comments, please use our discussion forum (available from the Help menu) or send us an email to hyperbrowser-requests@usit.uio.no‎. We are happy to respond and welcome collaborative projects that help further develop our system. If you encounter any bugs, please let us know via an email to hyperbrowser-requests@usit.uio.no.

If you register as a user, you will be able to subscribe to the hyperbrowser-requests@usit.uio.no mailing list, which serves for distribution of system-related announcements and administrative messages. A backlog of announcements is available from the Help menu. The Help menu also contains citation information.

This project is being developed by the Norwegian bioinformatics community as an open-source project under the GPL license v3, supported by various national and local bodies.

For further information on Norwegian bioinformatics, please visit www.bioinfo.no.
