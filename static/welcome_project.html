hGSuite HyperBrowser: A web platform for high dimensional genomic data analysis

description

Multi dimensional genomic data

Multidimensional genomic data can be presented as a matrix, with each row representing a genomic track and the column containing the features of the track. A collection of features of a track typically represents a cell type, mutation type, experimental condition and so on. These collections are also known metadata of the tracks. Here we present the multi dimensional genomic data in a GSuite format, that entitles each track along with its metadata. To know more about GSuite format please read GSuite specification

In the example below, each row represents a genomic tracks and its set of characteristics in each column (i.e. genotype and mutation type). This representation of the data allows for effective analysis of single genomic track collections as well as analysis within groups of genomic tracks.

The data can be grouped based on the question of interest, enabling easier analysis. For example if we are interested in statistical analysis like average number of genomic elements per defined group, where the group is defined by the character genotype (look at the left figure) or it can also be a group defined in combination with genotype and mutation type (look at the right figure).

In the example below, we calculated the average number of genomic elements for each tracks and calculated the average based on the group, eta and iota respectively. For the groups defined by the combinations of genotype and mutation type, we calculated the number of genomic elements for track 1 and 2 separately and calculated the average based genotype and mutation types like eta-CA group, eta-CG, iota-CA, iota-CT. For the groups defined by the combinations of genotype and mutation type, we calculated a number of genomic elements for tracks 1-2 separately and averaged them out (eta-CA group), and we did the same for the other groups (eta-CG, iota-CA, iota-CG, iota-CT).

Why do you want to use it?

The hGSuite HyperBrowser web platform is a set of independent tools that can be used in combination to perform an entire analysis starting from data preparation to result exploration. It offers an easy straight forward way to manage and analyse your data collection. At the core of the system is GSuite, a simple tabular format representing genomic track collection along with related metadata. hgSuite works on the metadata of such tracks. The figure below shows a pipeline used to analyse a collection of genomic tracks based on metadata of interest. The pipeline can be divided into four major steps.
  1. Data - A user can upload their own data or the data can be downloaded from external repositories using tools 'Create a GSuite from an integrated catalog of genomic datasets' and 'Create a remote GSuite from a public repository'. Data on a local computer can be organized into a hierarchical folder structure and uploaded to the web server as a tar or zip file (using tool 'Create a GSuite from an archive (Zip/tar) in history'). A hGSuite can be created from an uploaded tar file (using tool 'Create a GSuite from an archive (Zip/tar) in history'), where the folder structure of uploaded data will be reflected as metadata columns in the hGSuite. To rename metadata columns in the hGSuite can be used tool 'Modify metadata in hGSuite'. To add metadata we can use tool 'Add a metadata column in hGSuite'.
  2. Data pre-processing - Within the web platform, collections of datasets are represented in the GSuite format, which isi.e. a simple tabular text format that includes metadata for each track in a given collection. The platform includes tools for preprocessing the GSuite 'Preprocess a GSuite for analysis'.
  3. Statistical analysis and visualization - Analysis are provided within collection of genomic tracks (using tool 'Compute Pivot table for hGSuite') as well as pairwise combination of genomic tracks (using tool 'Compute Pivot table for relations between hGSuites'). They are restricted by the their metadata where hGSuites requires the metadata to be match. 'Create or modify hierarchy of hGSuite' tool is used to specify the metadata.
  4. Exploration of analysis results - The results of analysis for each genomic track is represented as a data cube (multidimensional array of values). A data cube is an arrangement of data suitable for analysis from different perspectives through operations like slicing, dicing, pivoting, and aggregation. We use the metadata of the genomic tracks based on the user input to carry out the above operation on the data cube. The results of that would be a two dimensional array, that would enable the user to draw overall conclusions in a time efficient manner. The tool 'Compute Pivot table for hGSuite' allows user to compute statistics for single hGSuite, while tool 'Compute Pivot table for relations between hGSuites' is used to represent results as data cube for two HGSuites.

Introduction

hgSuite can be used effectively on your data only if you can group them according to the meta information. Below are few examples of analysis done using different types of data ranging from self-uploaded to the data downloaded from public repo. These examples will help you understand how grouping works and helps in displaying results. Below are the examples to understand the tool better.

If you want to know more about tools itself go to tab called: Tools, if you want to know more about The Genomic HyperBrowser got to tab: Technical details.

1. Explore DNA repair enzymes and mutations (i.e knockout mouse data)

Our goal in this project is to explore the nature of DNA repair enzymes and mutations in knockout mouse data. We received 30 genomic tracks from our collaborators, these are called query tracks. The quantitative overview of the tracks such as count statistics of base-pair coverage per genomic track, segment distance average per genomic track and the local measurements such as coverage proportion per track per chromosome and counts per track per chromosome. The reference genome was downloaded from different databases such as ENCODE and UCSC and the different pairwise similarity measure analysis was repeated for every single query and reference track combination. This set of analysis produces several summary statistics and results that is informative, but not easily interpretable Grouping of the query tracks based on the metadata was used. Firstly, every query track was characterized by different mutation type (e.g. C>A, C>T, C>G, A>T, A>G, A>C, T>A etc), and therefore the query tracks were divided into two smaller tracks based on mutation types, resulting in 360 new genomic query tracks( 30 X 12 = 360 ). This shows that the data is multi dimensional, with two possible levels. The first level of the data was specified by different DNA repair enzymes, while the second level was identified by type of mutations. We ended up with 7 individual groups in the first level and 12 groups in the second level, and at this point we could start again with our analysis.

So we decided to make the grouping based on the meta levels.
